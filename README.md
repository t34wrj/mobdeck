# Mobdeck Mobile App

Mobdeck is a React Native mobile application that provides seamless access to your self-hosted [Readeck] (https://readeck.org/en/) instance. Designed for self-hosting enthusiasts, it offers offline-first article reading, background synchronization, and Android share integration for a complete mobile reading experience.

## Installation

### For End Users

Choose your preferred installation method:

#### Option 1: Obtainium (Recommended)

Obtainium provides automatic updates directly from GitHub releases:

1. **Install Obtainium**:
   - Download from [F-Droid](https://f-droid.org/) (recommended)
   - Or get APK from [Obtainium releases](https://github.com/ImranR98/Obtainium/releases)

2. **Add Mobdeck to Obtainium**:
   - Open Obtainium app
   - Tap "+" to add new app
   - Enter: `https://github.com/t34wrj/mobdeck`
   - Tap "Add App" and install

3. **Enable automatic updates**:
   - Configure check interval (recommended: 24 hours)
   - Enable background update checks
   - Set to WiFi-only if preferred

#### Option 2: Direct APK Download

1. **Download APK**:
   - Visit [Mobdeck Releases](https://github.com/t34wrj/mobdeck/releases)
   - Download latest `app-release.apk`
   - Verify SHA256 hash (provided in release notes)

2. **Install APK**:
   - Enable "Install from unknown sources" in Android Settings
   - Open downloaded APK file
   - Grant installation permissions
   - Complete installation

### Initial Setup

#### Bearer Token Authentication

Bearer tokens provide secure API access to your Readeck instance. Each token should be unique per device for security.

**Step 1: Generate Token in Readeck**

1. **Access Readeck Web Interface**:
   - Open your Readeck server URL in a web browser
   - Log in with your Readeck account credentials
   - Ensure you have administrative access to create API tokens

2. **Navigate to API Tokens**:
   - Click **Settings** in the main navigation
   - Select **API Tokens** from the settings menu
   - You'll see a list of existing tokens (if any)

3. **Create New Token**:
   - Click **"Create Token"** or **"+ Add Token"** button
   - **Name**: Enter descriptive name (e.g., "Mobdeck Mobile - [Your Device]")
   - **Description**: Add optional details (e.g., "Samsung Galaxy S23 - Primary")
   - **Permissions**: Select the following required permissions:
     - ‚úÖ **Read Articles** - View and download articles
     - ‚úÖ **Write Articles** - Add new articles via share integration
     - ‚úÖ **Manage Labels** - Create and assign article labels
   - **Expiration**: Set to "Never" or choose long-term duration

4. **Copy and Secure Token**:
   - Click **"Generate Token"**
   - **IMMEDIATELY COPY** the generated token (usually 32+ characters)
   - Token format: `abcd1234efgh5678ijkl9012mnop3456` (example)
   - ‚ö†Ô∏è **Important**: Save token securely for mobile configuration

**Step 2: Configure in Mobdeck**

1. **Open Mobdeck Authentication Screen**:
   - Launch Mobdeck app
   - Enter server URL and bearer token in the authentication form

2. **Enter Server Details**:
   - **Server URL**: Enter your complete Readeck URL
   - **Bearer Token**: Paste the copied token (no spaces)

3. **Test and Verify Connection**:
   - Tap **"Test Connection"** button
   - Wait for verification (may take 5-15 seconds)
   - Success indicators:
     - ‚úÖ "Connection Successful"
     - Server version displayed
     - Article count shown
   - If failed: Check troubleshooting section below

4. **Save Configuration**:
   - Tap **"Save Configuration"**
   - Token is encrypted and stored in Android Keychain
   - Configuration persists across app restarts

**Security Best Practices**:

- üîê Use unique tokens per device
- üìù Use descriptive token names for easy management
- üîÑ Rotate tokens periodically (every 6-12 months)
- üóëÔ∏è Revoke tokens for lost/replaced devices
- üö´ Never share tokens between users or apps
- üîí Use HTTPS servers when possible for token transmission

#### 3. Advanced Configuration Options

**Sync Preferences:**

_Frequency Options:_
Enter sync interval in minutes (e.g., 15 for every 15 minutes, 60 for hourly, 1440 for daily)

_Network Configuration:_

- ‚úÖ **WiFi Only**: Toggle to restrict sync to WiFi networks only

**Content Settings:**

## Troubleshooting

### Common Issues

#### Installation Problems

**APK Won't Install:**

- ‚úÖ Enable "Install from unknown sources" for browser/file manager
- ‚úÖ Ensure sufficient storage space (500MB+ recommended)
- ‚úÖ Disable Play Protect temporarily if it blocks installation
- ‚úÖ Try downloading APK again if corrupted
- ‚úÖ Check Android version compatibility (Android 7.0+ required)

**Obtainium Not Detecting Updates:**

- ‚úÖ Check internet connection
- ‚úÖ Refresh app list (pull down to refresh)
- ‚úÖ Verify repository URL: `https://github.com/t34wrj/mobdeck`
- ‚úÖ Check background app permissions for Obtainium
- ‚úÖ Update Obtainium to latest version

#### Authentication Issues

**Cannot Connect to Server:**

- ‚úÖ **URL Format**: Verify server URL includes protocol (`http://` or `https://`)
- ‚úÖ **Browser Test**: Test server access in mobile browser first
- ‚úÖ **Port Check**: Include port number if using non-standard port (e.g., `:8000`)
- ‚úÖ **Network Access**: Ensure server is accessible from mobile network
- ‚úÖ **DNS Resolution**: Try using IP address instead of hostname
- ‚úÖ **Firewall**: Check if firewall blocks mobile device IP range

**"Authentication Failed" Error:**

- ‚úÖ **Token Format**: Verify token is alphanumeric, 32+ characters
- ‚úÖ **Copy/Paste**: Check for extra spaces when copying token
- ‚úÖ **Permissions**: Ensure token has read/write permissions in Readeck
- ‚úÖ **Expiration**: Verify token hasn't expired in Readeck settings
- ‚úÖ **Regenerate**: Create new token if authentication continues to fail
- ‚úÖ **Case Sensitivity**: Tokens are case-sensitive - copy exactly

**"Server Not Reachable" Error:**

- ‚úÖ **Network Connectivity**: Test with other apps requiring internet
- ‚úÖ **Server Status**: Verify Readeck server is running and responsive
- ‚úÖ **VPN/Proxy**: Try connecting with/without VPN
- ‚úÖ **Different Network**: Test on different WiFi or mobile data
- ‚úÖ **Server Logs**: Check Readeck server logs for connection attempts

#### Sync Issues

**Articles Not Syncing:**

- ‚úÖ **Manual Sync**: Try "Pull to Refresh" to trigger manual sync
- ‚úÖ **Network Settings**: Verify sync allows current network type (WiFi/Cellular)
- ‚úÖ **Background Sync**: Check if background sync is enabled in settings
- ‚úÖ **Battery Optimization**: Disable battery optimization for Mobdeck
- ‚úÖ **Storage Space**: Ensure sufficient device storage (>100MB free)
- ‚úÖ **Token Validity**: Verify authentication token is still valid

**"Sync Failed" Error:**

- ‚úÖ **Connection Test**: Test server connection in app settings
- ‚úÖ **Restart App**: Force close and reopen Mobdeck
- ‚úÖ **Clear Cache**: Clear app cache in Android settings
- ‚úÖ **Network Quality**: Ensure stable internet connection
- ‚úÖ **Server Load**: Check if server is under heavy load

**Sync Taking Too Long:**

- ‚úÖ **Network Speed**: Test internet speed with speed test app
- ‚úÖ **Batch Size**: Reduce sync batch size in advanced settings
- ‚úÖ **Image Downloads**: Disable image downloading temporarily
- ‚úÖ **WiFi Connection**: Switch to WiFi for faster sync
- ‚úÖ **Server Performance**: Check server resource usage

**Background Sync Not Working:**

- ‚úÖ **Battery Optimization**: Disable for Mobdeck in Android settings
- ‚úÖ **Background App Refresh**: Enable in Android settings
- ‚úÖ **Data Saver**: Add Mobdeck to unrestricted apps
- ‚úÖ **Doze Mode**: Ensure app isn't affected by Android Doze mode
- ‚úÖ **Sync Schedule**: Verify sync interval is properly configured

#### Connection Issues

**Network Timeout Errors:**

- ‚úÖ **Signal Strength**: Ensure strong WiFi or mobile signal
- ‚úÖ **Network Speed**: Test connection speed with other apps
- ‚úÖ **Timeout Settings**: Increase network timeout in advanced settings
- ‚úÖ **Server Response**: Check if server responds to simple HTTP requests
- ‚úÖ **Switch Networks**: Try different WiFi or mobile data connection

**SSL/TLS Certificate Errors:**
